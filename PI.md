# Visema — Architecture Overview

> Auto-generated by pi-md skill. Last updated: 2026-06-06

## Quick Summary

Visema is a local Twitch bot that lets viewers spend Channel Points to display a GIF or play an audio clip on stream via an OBS Browser Source overlay. It listens for Channel Points redemptions via Twitch EventSub, validates the request (GIF URL against Giphy CDN domains, or sound name against a local `sounds/` index), enqueues it in a unified FIFO queue, and broadcasts the payload to the OBS overlay via WebSocket. The overlay renders GIFs with fade-in/out animations and plays audio clips with completion acknowledgment back to the queue worker. The bot uses a two-account design: a broadcaster account for EventSub redemptions and a bot account for chat messages.

## Module Descriptions

### Root-level items

#### `.env`
Stores Twitch OAuth credentials and access/refresh tokens for both broadcaster and bot accounts. Populated automatically by `auth.py` on first run. Never committed to version control.

#### `AGENTS.md`
Agent instructions document describing the project's architecture, conventions, and design decisions for AI coding assistants.

#### `config.yaml`
Runtime configuration file (safe to commit). Defines Twitch channel names, reward names, GIF validation rules (allowed domains, display duration, cooldown, queue size), audio settings (sounds directory, allowed extensions, volume), overlay server port and position, and chat command response strings.

#### `requirements.txt`
Python dependencies: `twitchAPI` (Twitch API client with EventSub support), `fastapi` (HTTP/WebSocket server), `uvicorn[standard]` (ASGI server), `websockets` (WebSocket support), `python-dotenv` (.env loading), `pyyaml` (config.yaml parsing).

#### `sounds/`
Local audio library directory. Streamer populates it with MP3/OGG/WAV files. On startup, `validator.py` scans this directory and builds an in-memory lookup index. Served as static files by FastAPI at `/sounds/`.

### `visema/`

#### `__init__.py`
Empty package marker.

#### `main.py`
CLI entrypoint. Parses `--setup` and `--verbose` arguments, loads settings, builds the sounds index, starts the FastAPI server in a background thread, initializes the media queue worker, authenticates both Twitch accounts, resolves channel IDs, and starts the EventSub listener and chat listener as concurrent asyncio tasks. Handles graceful shutdown on Ctrl+C.

### `visema/media/`

#### `__init__.py`
Empty package marker.

#### `queue.py`
Unified FIFO queue (`MediaQueue`) for both GIF and audio events. The worker coroutine dequeues items, broadcasts them via WebSocket to the overlay, then waits for completion: audio events await an `audio_done` ack from the overlay (with a 30s timeout fallback), GIF events complete automatically. After each item, the worker applies a configurable cooldown before processing the next. Supports `clear()` to drain pending items.

#### `validator.py`
GIF URL validation and audio name resolution. `build_sounds_index()` scans the `sounds/` directory and builds a `{stem: Path}` lookup dict. `validate_gif()` checks that a URL uses an allowed Giphy CDN domain (rejects bare `giphy.com` page URLs). `validate_audio()` normalizes input (lowercase, strip punctuation, spaces→underscores) and looks it up in the sounds index. `get_sound_names()` returns the sorted list of available sound names.

### `visema/overlay/`

#### `index.html`
OBS Browser Source page. Loads `overlay.css` and `overlay.js` from the `/static/` mount. Contains a single `#overlay-container` div as the rendering target.

#### `overlay.css`
Transparent background styles for OBS integration. Defines `.gif-image` with opacity transitions (fade in/out) and five position presets (`center`, `bottom-left`, `bottom-right`, `top-left`, `top-right`) applied via class on the container.

#### `overlay.js`
Receives WebSocket messages from the server. For `gif` messages: creates an `<img>` element, applies fade-in, holds for the configured duration, then fades out and removes the element. For `audio` messages: creates an `<Audio>` element, sets volume, plays it, and sends an `audio_done` ack back to the server when playback ends (or on a 30s timeout fallback). Reconnects to the WebSocket on disconnect with exponential backoff (up to 10s).

#### `app.py`
FastAPI application factory. Mounts `/static` for overlay CSS/JS files and `/sounds` for the audio library. Serves the overlay page at `GET /overlay`. Provides a WebSocket endpoint at `/ws` that delegates connection management to `ws_manager`. Includes a `/health` health-check endpoint.

#### `ws_manager.py`
`WebSocketManager` singleton that tracks active OBS Browser Source connections. `broadcast()` sends JSON payloads to all connected clients and cleans up dead connections automatically. `handle_client_ack()` forwards ack messages (like `audio_done`) to the queue worker via a module-level callback set by `queue.py`.

#### `auth.py`
OAuth2 authorization for both broadcaster and bot accounts. Loads existing tokens from `.env`, falls back to browser-based OAuth flow if tokens are missing/expired. Registers an auto-refresh callback (`_user_auth_refresh_callback`) that persists renewed tokens back to `.env`. Broadcaster scopes: `channel:read:redemptions`, `channel:manage:redemptions`. #### `chat.py`
IRC-based chat command listener using the bot account. Registers handlers for `!gif` (how to find Giphy direct links), `!sound` (how to use the sound redeem), and `!soundlist` (auto-generated list of available sounds from the sounds index). All commands are read-only — they do not interact with the queue or overlay.

EventSub WebSocket listener for `channel.channel_points_custom_reward_redemption.add` events. On startup, resolves reward IDs from the Twitch API. On each redemption, routes to `_handle_gif()` or `_handle_audio()` based on reward ID. Each handler validates the input, enqueues it, and either fulfills the redemption (if valid) or cancels it with a refund and chat message (if invalid). Includes re-auth retry logic for 401 errors.

## Data Flow

### Entity Lifecycles

#### Redemption Event (GIF)
- **Create/Source:** Viewer redeems "Show a GIF" on Twitch → EventSub WebSocket fires `channel.channel_points_custom_reward_redemption.add` event
- **Read/Access:** `eventsub.py::RedemptionHandler.on_redemption()` extracts user, reward ID, and input URL
- **Transform:** `validator.validate_gif()` checks URL scheme, hostname against allowed Giphy CDN domains → returns URL or None
- **Update:** If valid: `_fulfill_redemption()` marks redemption as FULFILLED via Twitch API. If invalid: `_cancel_redemption()` refunds points and posts reason in chat
- **Consume/Output:** Valid item enqueued in `MediaQueue` → worker broadcasts `{type: "gif", url, duration}` via WebSocket → `overlay.js::showGif()` renders `<img>` with fade-in/out

#### Redemption Event (Audio)
- **Create/Source:** Viewer redeems "Play a Sound" on Twitch → EventSub WebSocket fires redemption event
- **Read/Access:** `eventsub.py::RedemptionHandler.on_redemption()` extracts user, reward ID, and sound name
- **Transform:** `validator.validate_audio()` normalizes input (lowercase, strip punctuation, spaces→underscores) and looks up in `_sounds_index` → returns Path or None
- **Update:** If found: `_fulfill_redemption()` marks as FULFILLED. If not found: `_cancel_redemption()` refunds points
- **Consume/Output:** Valid item enqueued in `MediaQueue` → worker broadcasts `{type: "audio", src, volume}` via WebSocket → `overlay.js::playAudio()` plays audio → on "ended", sends `{ack: "audio_done"}` back → queue worker receives ack via `ws_manager::handle_client_ack()` → `MediaQueue._on_ack()` → `_audio_done_event.set()` → cooldown starts

#### Sound Index
- **Create/Source:** `sounds/` directory on disk (streamer populates manually)
- **Transform:** `validator.build_sounds_index()` scans directory on startup → builds `{stem: Path}` dict
- **Read/Access:** `validator.validate_audio()` reads from `_sounds_index`; `validator.get_sound_names()` returns sorted keys for `!soundlist`
- **Consume/Output:** Used by EventSub handler for audio redemption validation; used by chat handler for `!soundlist` command

#### OAuth Tokens
- **Create/Source:** Browser OAuth flow via `auth.py::_do_browser_auth()` (first run or expired tokens)
- **Transform:** Tokens persisted to `.env` via `_write_env()`
- **Read/Access:** `auth.py::_authenticate_account()` loads from `.env` via `_read_env()`
- **Update:** Auto-refresh callback `_user_auth_refresh_callback()` writes new tokens to `.env` on every refresh
- **Consume/Output:** `broadcaster_client` used for EventSub and redemption status API calls; `bot_client` used for chat messages

### Request/Response Flows

- **GIF Redemption:** Twitch EventSub → `RedemptionHandler.on_redemption()` → `validate_gif()` → `MediaQueue.enqueue()` → `_fulfill_redemption()` → chat notification → queue worker → `WebSocketManager.broadcast()` → OBS overlay
- **Audio Redemption:** Twitch EventSub → `RedemptionHandler.on_redemption()` → `validate_audio()` → `MediaQueue.enqueue()` → `_fulfill_redemption()` → chat notification → queue worker → `WebSocketManager.broadcast()` → OBS overlay → audio playback → `{ack: "audio_done"}` → `MediaQueue._on_ack()` → cooldown
- **Chat Commands:** Twitch IRC → `ChatCommandHandler.on_gif/on_sound/on_soundlist()` → `bot_client.send_chat_message()` → chat response

### Event & Queue Flows

- **EventSub events:** Twitch → `EventSubWebsocket` → `RedemptionHandler.on_redemption()` → validation → `MediaQueue`
- **Queue processing:** `MediaQueue._worker()` → dequeue → `WebSocketManager.broadcast()` → (audio: wait for ack) → cooldown → next item
- **Audio ack:** OBS overlay → WebSocket `{ack: "audio_done"}` → `WebSocketManager.handle_client_ack()` → `_ack_callback` → `MediaQueue._on_ack()` → `_audio_done_event.set()`

### External Data Flows

- **Inbound:**
  - Twitch EventSub WebSocket → redemption events
  - Twitch IRC → chat commands
  - Giphy CDN → GIF images (loaded directly by overlay from CDN URL)
  - Local `sounds/` directory → audio files (served as static files by FastAPI at `/sounds/`)

- **Outbound:**
  - Twitch API → redemption status updates (fulfill/cancel)
  - Twitch chat → bot messages (confirmations, errors, command responses)
  - WebSocket → overlay payloads (gif/audio events)
  - FastAPI static mount → audio files served to OBS Browser Source

## Architecture Notes

- **Two-account design:** Broadcaster account handles EventSub redemptions and API calls; bot account handles chat messages. This separation prevents the bot from interfering with the streamer's identity.
- **Unified queue:** GIF and audio events share a single `asyncio.Queue` with shared max size and cooldown. This prevents resource contention and ensures orderly processing.
- **Audio ack pattern:** Unlike GIFs (which auto-complete via CSS timeout), audio events require the overlay to signal completion via WebSocket ack. The queue worker awaits this ack (with a 30s timeout fallback) before starting the cooldown timer.
- **No external API keys:** GIF validation uses only URL domain checking against Giphy CDN — no API key needed. Audio is entirely local.
- **Zero external CDN dependencies in overlay:** The OBS Browser Source page loads CSS and JS from the local FastAPI server. GIFs are loaded from Giphy's CDN (the URL is part of the event payload), and audio is served from the local `/sounds/` mount.
- **Re-auth resilience:** Both EventSub and auth modules handle 401 errors by retrying with re-authentication, following the pattern from the twitch-tts-git family of bots.
