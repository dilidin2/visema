Features

Everything your stream needs,
nothing it doesn't

Built for streamers who want a clean, local-first experience with zero external dependencies.

🎬

GIF Redemptions

Viewers paste a direct Giphy CDN link and the GIF plays on your OBS overlay with smooth fade-in/out animations. Invalid URLs are auto-cancelled with a point refund.

🔊

Audio Redemptions

Viewers type a sound name and the bot plays it from your local sounds/ folder. No external APIs — just drop MP3, OGG, or WAV files and you're set.

🔐

Two-Account Design

Your broadcaster account handles EventSub redemptions. A separate bot account sends chat messages. Clean separation, no identity mixing.

⚡

Unified Queue

GIFs and audio share a single FIFO queue with configurable cooldown and max size. No overlapping, no chaos — orderly processing every time.

🖥️

OBS Browser Source

The overlay connects via WebSocket to your local server. Add it as a Browser Source in OBS and it just works — no manual configuration in OBS needed.

💬

Chat Commands

!gif for help, !sound for instructions, !soundlist for the full sound list. All responses are configurable in config.yaml.

How It Works

From redemption to overlay
in four steps

A viewer spends Channel Points — here's what happens behind the scenes.

1

Viewer redeems

A viewer spends Channel Points on your stream, providing a GIF URL or sound name as input. Twitch fires an EventSub event to Visema.

2

Validation

GIF URLs are checked against Giphy's CDN domains. Sound names are normalized and looked up in the local index. Invalid requests are cancelled with a refund.

3

Queue & broadcast

Valid requests enter the FIFO queue. The worker dequeues and broadcasts the payload via WebSocket to the OBS overlay.

4

Overlay plays

The OBS Browser Source renders the GIF with fade animations or plays the audio clip. Audio sends an ack back when done, triggering the cooldown.

Setup

Up and running
in minutes

Install dependencies, configure two files, and you're live.

Terminal 
# Install uv (dependency manager)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Clone & set up
git clone https://github.com/dilidin2/visema.git
cd visema

uv venv
source .venv/bin/activate
uv pip install -r requirements.txt

# Configure credentials
cp .env.example .env
cp config.yaml.example config.yaml

# First run — authorizes both accounts via browser
python -m visema.main --setup

# Normal run
python -m visema.main
